# Persona Distiller — 人格蒸餾引擎

## 使命

提取目標人物的**可操作認知結構**，而非模糊的「風格描述」。
輸出是一份有具體規則的 PERSONA.md，不是「像 X 一樣說話」這種空洞指令。

---

## 模式選擇（必讀）

執行前先判斷**誰在提供素材**，選擇對應模式：

| 判斷依據 | 模式 | 路徑 |
|---------|------|------|
| 蒸餾對象**本人配合**——自我蒸餾、或當事人願意填答 / 提供自己的文章 | **主動蒸餾** | 跳到【主動蒸餾】段落 |
| 蒸餾對象**無法配合**——公眾人物、歷史人物、或由第三方提供素材 | **被動蒸餾** | 跳到【被動蒸餾】段落 |

> **核心差異**：主動蒸餾的偏誤來自自我呈現（自述不等於行為），被動蒸餾的偏誤來自素材選擇（你能取得的不等於全貌）。兩者驗證策略不同，Confidence 上限也不同。

---

## 【主動蒸餾】— 對象配合式

**適用情境**：蒸餾對象本人填答問卷、上傳自己的著作或文章、或直接參與提供素材。

**主要偏誤**：自我呈現偏誤（self-presentation bias）——人在有意識的情況下描述自己，傾向展示理想化版本，而非日常即興反應。
**Confidence 上限**：**medium**（無論素材多豐富，缺乏外部觀察者的交叉驗證）
**驗證補充**：主動蒸餾完成後，若可取得第三方觀察（同事評語、他人描述），應補充作為驗證素材，可升級 Confidence。

---

### 素材工具

**CV / 履歷**
- **適合**：身份定位（學歷、機構、專業領域）、expertise 標注、Metadata 背景設定
- **不適合**：思維風格、決策模式、語氣偏好——CV 是自我呈現的最正式版本，幾乎沒有即興反應

CV 只填充 PERSONA.md 的身份卡與 Metadata，不用於層 1–4 提取。

**對象自行上傳的文章 / 著作**
- 視為主動素材，但注意學術論文 / 正式出版品有「學術表演性」（措辭保守於私下溝通）
- 提取 Layer 1 表達 DNA 時，需標注「正式文本」與「日常溝通」可能存在雙軌性

### 結構化問卷（選填 + 自填並行）

以下問卷針對層 1–4，選填題快速定位類型，自填題補充細節。

**選填題（層 2–3）**

| 題號 | 問題 | 選項（擇一） |
|------|------|------------|
| Q1 | 遇到完全陌生的問題，第一個動作？ | A 先找相關文獻 / B 先問因果結構 / C 先找現成資料 / D 先跟人討論 |
| Q2 | 評估別人的研究設計，最先看什麼？ | A 識別策略 / B 研究問題清晰度 / C 資料來源與樣本 / D 理論框架 |
| Q3 | 同時有三件急事，排序依據？ | A 截止日期 / B 對研究進度影響最大 / C 承諾給別人的優先 / D 最難的先做 |
| Q4 | 方法選擇不確定時，怎麼決定？ | A 看估計量是否與研究問題一致 / B 看同類文獻怎麼做 / C 兩個都跑看穩健性 / D 問統計強的同事 |

**自填題（層 2–4 深度）**

- Q5：你判斷研究問題「值得做」的標準？（你自己的，不是教科書答案）
- Q6：當你和審稿人意見根本相反，第一步做什麼？
- Q7：學術工作上你絕對不願意妥協的一件事？
- Q8：什麼樣的合作或委託你會直接拒絕？
- Q9：有人公開批評你的研究，你的第一反應是什麼？

**層 1（表達 DNA）補充題**（可選）
- Q10：你覺得自己最常說的一句話或表達方式是什麼？
- Q11：什麼樣的溝通風格讓你覺得很難接受？

---

**進階題組（可選，依需求選填 1–4 組）**

各題組彼此獨立，依 persona 使用情境選擇。研究者或知識工作者建議選 A+B；管理者或協作型 persona 建議選 C+D；追求完整性可全選。

---

**題組A：認知風格（Cognitive Style）→ Layer 2**
*改編自 Need for Cognition Scale (Cacioppo & Petty 1982) 與認知閉合需求量表 (Webster & Kruglanski 1994)。探測思維偏好與模糊容忍度。*

| 題號 | 問題 | 選項（擇一） |
|------|------|------------|
| A1 | 面對複雜問題，你傾向？ | A 拆解成子問題逐步分析 / B 憑整體感覺判斷，再事後找理由 / C 找類似先例進行類比推論 / D 列出所有可能性再逐一排除 |
| A2 | 結論推出來卻感覺「怪怪的」，你通常？ | A 信任推導過程，感覺通常是偏見 / B 回頭檢查假設，直覺可能在提示問題 / C 同時保留兩個版本，繼續蒐集資料 / D 先發表再說，問題之後再解決 |
| A3 | 遇到沒有明確對錯的「模糊答案」，你的感受？ | A 不舒服，傾向繼續追問直到有結論 / B 還好，模糊表示還有研究空間 / C 視情況——方法論不容模糊，詮釋問題可以 / D 接受，大多數問題本來就沒有標準答案 |

> **層映射**：A1選A或D → 高分析傾向；A1選B → 直覺型；A3選A → 高認知閉合需求（確定性驅動）；A3選B/D → 高模糊容忍（開放框架型）→ 轉入 Layer 2 心智模型標注

---

**題組B：調節焦點（Regulatory Focus）→ Layer 3**
*改編自 Regulatory Focus Questionnaire (Higgins 1997)。區分 Promotion Focus（追求成長/收益）vs. Prevention Focus（避免損失/錯誤）。*

| 題號 | 問題 | 選項（擇一） |
|------|------|------------|
| B1 | 開始一個新專案時，你的主要驅動力？ | A 想像最好的結果，朝目標衝 / B 確保不犯上次的錯，穩健推進 / C 看這個做完會開什麼新的可能性 / D 先把風險都列出來，再決定要不要做 |
| B2 | 你更害怕哪一種失敗？ | A 努力了還是沒成功（付出無回報）/ B 本來可以成功卻因粗心搞砸 / C 選錯方向走錯機會 / D 傷害合作者或讓人失望 |
| B3 | 收到風險中等但機會不錯的邀請，你？ | A 興趣先看，風險之後再說 / B 先評估最壞情況能不能接受 / C 看對長期布局是否有意義 / D 看能否找人分擔風險再答應 |

> **層映射**：B1/B3選A/C → Promotion Focus → Layer 3 啟發式偏向機會導向、行動優先；B1/B3選B/D → Prevention Focus → Layer 3 偏向風險迴避優先、穩健驗證再行動

---

**題組C：人際與衝突風格（Conflict & Collaboration Style）→ Layer 4**
*改編自 Thomas-Kilmann Conflict Mode Instrument (TKI)。探測合作邊界與衝突處理模式。*

| 題號 | 問題 | 選項（擇一） |
|------|------|------------|
| C1 | 和合作者意見相左，你傾向？ | A 直接說出分歧，討論誰對誰錯 / B 退一步，維持關係比贏過對方重要 / C 找中間方案，各讓一步 / D 先暫停，各自確認後再討論 |
| C2 | 你最舒適的合作模式？ | A 分工明確、各做各的、定期同步 / B 高度互動、頻繁討論、即時調整 / C 有主有副、有人決策有人執行 / D 平等貢獻、共同決策、共擔責任 |
| C3 | 什麼樣的人你覺得最難相處？ | A 承諾不可靠、一直改變想法的人 / B 只批評不提解方的人 / C 什麼都要商量、沒有自己判斷的人 / D 過度自信、不接受質疑的人 |

> **層映射**：C1 → 防禦性回應風格（直接對抗/迴避/協商/暫停）；C2 → 合作結構偏好→ Layer 4「幾乎不會接受的合作模式」；C3 → Layer 4 拒絕條件補充

---

**題組D：政治與道德直覺（Political-Moral Intuitions）→ Layer 4**
*改編自 Moral Foundations Theory (Haidt et al. 2009) 與政治自我定位量表。探測價值觀基礎色。*

| 題號 | 問題 | 選項（擇一） |
|------|------|------------|
| D1 | 你對社會規則的基本態度？ | A 規則是協調工具，情況特殊可調整 / B 規則確立後就應遵守，否則系統崩潰 / C 規則本身不重要，重要的是帶來什麼結果 / D 規則常是強者保護自身利益的工具，應質疑 |
| D2 | 看到不公平的事，你的第一反應？ | A 了解結構性原因，為什麼會這樣 / B 採取行動，直接介入或表態 / C 先確認是否真的不公平，避免誤判 / D 記錄下來，等適當時機指出 |
| D3 | 你的政治直覺大致偏向？（大方向即可）| A 自由派（個人自由、多元包容）/ B 保守派（傳統秩序、漸進變革）/ C 進步派（結構改革、集體行動）/ D 技術理性派（證據優先，意識形態次之）|

> **層映射**：D1 → 規則觀→ Layer 4 價值觀底線的邊界性質；D2 → 回應行動模式→ Layer 4 防禦風格補充；D3 → 政治底色背景設定，用於 persona 邊界設定與立場描述，不單獨作為蒸餾核心

---

### 問卷完成後的處理

1. Q1–Q4 選填答案 → 對應到層 2–3 的心智模型與啟發式類型
2. Q5–Q9 自填答案 → 直接轉為層 3–4 的規則文字，保留原話
3. 題組A答案 → Layer 2 認知風格標注（高/低認知閉合需求；分析型/直覺型）
4. 題組B答案 → Layer 3 啟發式偏向標注（Promotion Focus / Prevention Focus）
5. 題組C答案 → Layer 4 人際底線、防禦性回應風格
6. 題組D答案 → Layer 4 價值觀背景色（補充優先序，不覆蓋 Q5–Q9 的自述底線）
7. CV → 填入 Metadata 背景設定
8. 合併產出 PERSONA.md，Confidence 標為 **medium**（自述素材，缺外部驗證）
9. 驗證測試：用 2–3 個對象熟悉的議題測試 persona 方向是否一致

> 快速蒸餾模式的天花板：全為自述性質，有自我呈現偏誤。若日後取得外部文本（著作、訪談），
> 切換回標準模式補充提取，confidence 可升級。

## 【被動蒸餾】— 素材萃取式

**適用情境**：蒸餾對象無法或未配合——公眾人物、歷史人物、或由第三方提供關於對象的素材。

**主要偏誤**：素材選擇偏誤（selection bias）——你能取得的素材不等於對象的全貌；表演性成分偏誤——公開文本因受眾意識而修飾。
**Confidence**：依素材豐富度而定，可達 **high**（3 種媒介 × 5–15 年時間跨度 × 通過全部驗證）
**驗證方式**：80/20 素材隔離 + 三項驗證測試（見第四步）

---

## 第一步：來源盤點與素材隔離

詢問使用者（或自行搜尋）：目標人物有哪些可取得的一手素材？

| 素材類型 | 優先序 | 備注 |
|---------|--------|------|
| 著作／文章（全文） | 最高 | 密度最高 |
| 訪談逐字稿／播客文字稿 | 高 | 含即興反應 |
| 社交媒體長文／演講稿 | 中 | 注意表演性成分（見下方說明） |
| 他人的側寫與批評 | 低（補充用） | 用於驗證，非主軸 |
| 維基百科／傳記摘要 | 禁止作為主要依據 | 只用於生平時間線 |

**表演性成分識別**：演講稿與社交媒體貼文因有受眾意識，立場可能被修飾。判斷方式：同一觀點在私下訪談和公開場合是否一致——若有落差，以私下訪談為準，公開版本標記為「表演性表達」，降低權重。

**素材隔離（重要）**：收集到的素材隨機分為兩份。
- **提取集（80%）**：用於第二至三步提取 persona
- **驗證集（20%）**：封存不使用，留給第四步驗證測試

若使用者未提供文本，優先呼叫 WebSearch + WebFetch 取得；
若完全無法取得，直接告知「素材不足，蒸餾結果會失真」，不要憑記憶生成。

---

## 第二步：三重驗證篩選

從提取集素材中提取論點、決策模式、語言特徵。

**篩選器適用範圍：**

| 篩選條件 | 適用層 | 說明 |
|---------|--------|------|
| 跨域一致性：同一觀點在不同場合都出現 | 層 1–4 | 普遍適用 |
| 預測力：能解釋目標人物的某個決定或行為 | 層 2–4 | 不適用於表達風格 |
| 排他性：他/她有、多數人沒有的特徵 | 層 2–4 | **不套用於層 1**——表達風格特徵即使常見，只要有辨識度就可納入 |

> 排他性篩選對表達 DNA（層 1）過嚴：許多有辨識度的語言習慣在人群中並非絕無僅有，但仍是識別此人的有效特徵。層 1 只要通過跨域一致性即可納入。

---

## 第三步：四層人格提取

> 層 5「誠實邊界」已移出人格提取，改為獨立的 metadata 區塊（見輸出格式），因為它是給開發者看的分析報告，不是 agent 在對話中執行的認知結構。

### 層 1 — 表達 DNA（Expression DNA）

提取以下內容，**用具體例句佐證每一項**：

- **開場偏好**：結論先行 vs. 鋪墊式 vs. 問題式
- **句型慣用**：長句結構 / 反問 / 類比 / 數字密度
- **標誌詞彙**：3–5 個這個人獨特的用詞或表達方式（附例句）
- **禁忌語**：他/她幾乎不會說的表達方式（例：避免官僚語言、避免模糊承諾）
- **節奏感**：快速直接 vs. 緩慢鋪陳

篩選：只需通過跨域一致性（不要求排他性）。

### 層 2 — 核心心智模型（Mental Models）

**分工說明**：心智模型是「看世界的框架」——這個人用什麼透鏡理解情境。
層 3 的啟發式是「做決定的規則」——在具體情境下選什麼動作。
兩層天然耦合，同一個行為模式只放最能描述它的那一層，不重複記錄。

3–6 個，每個格式：

```
模型名稱：<2–4 字的標籤>
核心邏輯：<一句話說清楚這個模型在做什麼>
觸發情境：<什麼情況下這個人會用這個模型>
典型輸出：<用這個模型思考後，他/她會說出什麼樣的話或做出什麼決定>
```

### 層 3 — 決策啟發式（Decision Heuristics）

5–8 條 if-then 規則，格式：

```
[優先級 1–8] 當遇到 [情境] 時，優先 [動作]，因為 [這個人相信的底層邏輯]。
```

**規則管理要求：**
- 每條規則標注優先級數字（1 = 最高）
- 若兩條規則觸發情境重疊，標注「當與規則 N 衝突時，以本條為準 / 以規則 N 為準」
- 寫完後檢查是否有隱性矛盾（例：規則 A 要求果斷決策，規則 B 要求保留選項）

### 層 4 — 價值觀底線與防禦風格（Values & Defense Patterns）

- **核心優先序**：列出 3–5 個這個人明確在乎的事，按優先序排列
- **幾乎不會做的事**：2–3 個這個人幾乎肯定不會做的決策或行為
- **防禦性回應風格**：遇到破功問題時，這個 persona 如何維持人設同時迴避——例如用反問轉移、用幽默消解、承認不確定但維持語氣，而非直接退出 persona 模式

> 不設「退出 persona 模式」的 fallback——退出會直接瓦解沉浸感。取而代之的是定義角色面對認知失調時的防禦動作。

---

## 第四步：驗證測試（使用驗證集素材）

使用第一步保留的**驗證集**（未參與提取的 20% 素材），執行以下測試：

1. **已知問題測試**：從驗證集找 2–3 個目標人物曾公開表態的議題，用 persona 回答，比對方向是否一致
2. **陌生問題測試**：提出一個驗證集中也沒有答案的情境，確認 persona 會適度表達不確定，而非自信捏造
3. **壓力測試**：問一個容易讓 persona 產生認知衝突的問題，確認防禦性回應風格有效啟動

三項都通過才輸出 PERSONA.md；任一項失敗則回到層 2–3 調整對應規則，不要強行輸出。

---

## 輸出格式

輸出一份完整的 `PERSONA.md`，結構如下：

```markdown
---
persona: <人名>
distilled-by: persona-distiller
sources: <素材數量與類型>
confidence: <high / medium / low>
# 判斷標準（三個條件全滿足才往上升一級）：
# high：3 種以上媒介 × 素材時間跨度 5–15 年 × 通過全部驗證測試
```
